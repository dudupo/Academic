\begin{frame}
\frametitle{Majority}

\begin{block}{Alg.}
  Each bit looks at its neighbour checks, if the majority of them is unsatisfied, then it flip itself. 
\end{block}

\begin{block}{Justification.}
  Was suggested in Dennis. 
\end{block}

\begin{block}{Bottom line.}
  \begin{itemize}
    \item Fails to correct, even in the absence of noise. 
    \item Collisions.  
    \item Gives better result when requiring $(\frac{1}{2} + \mu)$-majority. 
  \end{itemize}
\end{block}

\end{frame}

\begin{frame}
\frametitle{Colors}

\begin{block}{Alg.}
  At the initialization, We divide the bits into subsets (colors). Bits of the same color don't share common check. Then, at the correction round $i$th only bits associated with the color $i \mod \textbf{COLORS}$ apply a correction. 
\end{block}

\begin{block}{Justification.}
  \begin{enumerate}
    \item No collisions, means that bits don't wait until they can read the syndrome.  
    \item The independence might help when coming to the math. 
  \end{enumerate} 
\end{block}

\begin{block}{Bottom line.}
  \begin{itemize}
    \item I took $32$ colors, and sample a random coloring. 
    \item Fails to correct even in the absence of noise. 
    \item Not tested yet when taking $(\frac{1}{2} + \mu)$-majority. 
  \end{itemize}
\end{block}

\end{frame}

\begin{frame}
\frametitle{Picking Random Pair}

\begin{block}{Alg.}
  Each bit pick two random neighbour checks, and then check if both of them are not satisfied. If so, then flip itself. 
\end{block}

\begin{block}{Justification.}
  With good probability, no too many collisions. Yet, all the bits participate in each round. 
\end{block}

\begin{block}{Bottom line.}
  \begin{itemize}
    \item Fails to correct even in the absence of noise. 
    \item Can we find an analog to $(\frac{1}{2} + \mu)$-majority? Maybe picking $3$? 
  \end{itemize}
\end{block}

\end{frame}

\begin{frame}
\frametitle{Swift Rule}

\begin{block}{Alg.}
  We define a north vector $\mathbf{1} = (1,1,1..1)$, Then each vertex looks only on the cells on its neighbourhood such that they are positive relative to $\mathbf{1}$.


  Then, the vertex suggest a correction over those cells, that decreases the local syndrome. 
\end{block}

\begin{block}{Justification.}
  \begin{enumerate}
    \item In the absence of noise, was proven to work. 
    \item Considered a generalization of Toom's rule.
  \end{enumerate}
\end{block}

\begin{block}{Bottom line.}
  \begin{itemize}
    \item Works in the absence of noise. 
    \item Simulation are too slow for side length of $n > 16$
    \item Don't have yet results for accumulating setting. 
  \end{itemize}
\end{block}

\end{frame}

\begin{frame}
\frametitle{Max Color}

\begin{block}{Alg.}
  We change the setting. We back to the setting that allows a non-nosy computation aside, yet we would like to limit the communication to constant number of bits.
\end{block}

\begin{block}{Justification.}
  \begin{enumerate}
    \item We don't have works about bounded communication fault tolerance.  
    \item From mathematical point of view, if the number of colors is $c$, then we can handle a fraction of $1/c$ of the dirty bits.
  \end{enumerate}
\end{block}

\begin{block}{Bottom line.}
  \begin{itemize}
    \item Works in the absence of noise. 
    \item 'Simulatable'. 
    \item Should be tried when vertices correct according to $(\frac{1}{2} + \mu )$ majority.  
  \end{itemize}
\end{block}
\end{frame}
